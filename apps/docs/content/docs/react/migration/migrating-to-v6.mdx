This page summarizes the current API. For full details see [Define](/docs/react/api-references/define) and [Hook](/docs/react/api-references/hook).

## defineStepper

- **Signature:** `defineStepper(...steps)` — only the steps array (or spread steps). No second config argument.
- **Returns:** `{ steps, utils, Scoped, useStepper, Stepper }`.

## useStepper

- **Config:** `useStepper({ initialStep?, initialMetadata? })` — optional. No `initial`, `mode`, or global transition callbacks.
- **Returns:** Stepper object with:
  - **State:** `all`, `current`, `isFirst`, `isLast`, `isTransitioning`, `metadata`
  - **Navigation:** `next()`, `prev()`, `goTo(id)`, `reset()`
  - **Step lookup:** `get(id)`
  - **Metadata:** `setMetadata(id, values)`, `getMetadata(id)`, `resetMetadata(keepInitial?)`
  - **Rendering:** `when()`, `switch()`, `match()`
  - **Transition hooks:** `onBeforeTransition(cb)`, `onAfterTransition(cb)` — call these to register callbacks; they run on every `next`/`prev`/`goTo`. Return `false` from the before callback to cancel.

## Scoped

- **Props:** `initialStep?`, `initialMetadata?`, `children` — no `initial` object or `statuses`.

## Step and current

- `stepper.current` is the **step object** (e.g. `{ id, title }`), not a wrapper with `.data` or `.status`.
- Use `stepper.all` for the list of steps (not `stepper.steps`).
- Use `stepper.get(id)` to get a step by ID (not `stepper.step(id)`).
- Step-level UI status (`active` / `success` / `inactive`) comes from primitives (e.g. `useStepItemContext().status`), not from a global status record.

## Core package

`@stepperize/core` exports types (`Step`, `Stepper`, `Utils`, `Get`) and pure helpers (`generateStepperUtils`, `getInitialStepIndex`, `getInitialMetadata`, `generateCommonStepperUseFns`, `updateStepIndex`). The React package uses these; you typically use `@stepperize/react` only.
