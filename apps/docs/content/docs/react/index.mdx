# Simplifying Multi-Step Processes in React

Stepperize is a versatile React library designed to streamline the creation of Stepper components. It empowers developers to build complex, multi-step processes with ease, allowing them to focus on business logic rather than implementation details.

## Key Features

- **Type-Safe Primitives**: Fully typed components for building accessible stepper UIs
- **Flexible Step Management**: Create linear or free-form navigation flows
- **Headless Component**: Full control over styling and UI
- **React Native Compatible**: Build steppers for web and mobile apps
- **Conditional Rendering**: Dynamically adjust content based on user input
- **State Management**: Built-in tools for handling step state, metadata, and status
- **Persistence**: Save and restore stepper state with built-in persistence
- **Async Support**: Handle async initialization and navigation callbacks

## Quick Start

```tsx
import { defineStepper } from "@stepperize/react";

const { Stepper, useStepper } = defineStepper([
  { id: "shipping", title: "Shipping" },
  { id: "payment", title: "Payment" },
  { id: "confirmation", title: "Confirmation" }
]);

function Checkout() {
  return (
    <Stepper.Root>
      {({ stepper }) => (
        <>
          <Stepper.List>
            {stepper.steps.map((stepInfo) => (
              <Stepper.Item key={stepInfo.data.id} step={stepInfo.data.id}>
                <Stepper.Trigger>
                  <Stepper.Indicator />
                  <Stepper.Title>{stepInfo.data.title}</Stepper.Title>
                </Stepper.Trigger>
                <Stepper.Separator />
              </Stepper.Item>
            ))}
          </Stepper.List>

          {stepper.switch({
            shipping: () => <ShippingForm />,
            payment: () => <PaymentForm />,
            confirmation: () => <Confirmation />
          })}

          <Stepper.Actions>
            <Stepper.Prev />
            <Stepper.Next />
          </Stepper.Actions>
        </>
      )}
    </Stepper.Root>
  );
}
```

## Core Concepts

### Thinking in Steps

Steppers break down complex workflows into manageable chunks, providing:

- Clear navigation between steps
- Visual progress indicators
- Improved user experience for multi-stage processes

**Common Use Cases**

- User onboarding
- Multi-page forms
- E-commerce checkout
- Product creation wizards
- Tutorial walkthroughs
- Surveys and questionnaires
- Booking systems

### Thinking in Flows

Stepperize goes beyond linear progressions, enabling:

- Conditional rendering
- Branching paths
- Dynamic form adjustments
- Personalized user experiences
- State-dependent UI changes
- Multi-path processes

### Step Information

Each step provides comprehensive information:

```tsx
const stepper = useStepper();

// Access current step
stepper.current.data;       // Step definition { id, title, ... }
stepper.current.status;     // "inactive" | "active" | "loading" | "error" | "success" | "skipped"
stepper.current.metadata;   // Custom data storage
stepper.current.isCompleted; // true if status is "success"

// Modify step state
stepper.current.setStatus("success");
stepper.current.setMetadata({ formData: values });
```

### Navigation

Multiple ways to navigate:

```tsx
// Basic navigation
stepper.next();
stepper.prev();
stepper.goTo("payment");
stepper.reset();

// With callbacks
stepper.beforeNext(async () => {
  const isValid = await validateForm();
  return isValid; // Return false to cancel
});
```
