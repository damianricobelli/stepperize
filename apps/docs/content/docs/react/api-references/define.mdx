`defineStepper` creates a stepper from your step definitions. You pass one object per step (as separate arguments). It returns typed `Stepper` (primitives), `Scoped`, `useStepper`, and `steps`.

## Basic usage

```tsx twoslash
import { defineStepper } from "@stepperize/react";

const { Stepper, Scoped, useStepper, steps } = defineStepper(
  { id: "shipping", title: "Shipping", description: "Enter your address" },
  { id: "payment", title: "Payment", description: "Payment details" },
  { id: "confirmation", title: "Confirmation", description: "Review order" }
);
```

Each step must have a unique `id`. You can add any custom properties (`title`, `description`, `icon`, etc.) for your UI.

## Return value

| Property | Type | Description |
| -------- | ---- | ----------- |
| `steps` | `Steps` | The array of step objects you passed |
| `useStepper` | `(config?) => Stepper` | Hook for state, navigation, lookup, flow, metadata. Optional config: `initialStep`, `initialMetadata` ([Hook](./hook)) |
| `Scoped` | `Component` | Provider for stepper context ([Scoped](./scoped)) |
| `Stepper` | `StepperPrimitives<Steps>` | Type-safe UI primitives ([Primitives](./primitives)) |
